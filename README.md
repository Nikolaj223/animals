# Лапа Надежды

Современный сайт приюта для кошек и собак. Проект собран так, как это обычно делают в рабочих frontend-проектах: HTML здесь используется только как точка входа для страницы, а интерфейс, логика и данные разложены по отдельным модулям.

В проекте уже есть:

- главная страница;
- страница `О приюте`;
- страница `Новости`;
- каталог животных;
- отдельные карточки животных;
- анкета `Хочу приютить`;
- страница `Помощь приюту` с формой;
- окно `Задать вопрос` с ответами из кода;
- JSON-хранилище заявок;
- админ-панель, которая показывает реальные записи из JSON;
- smoke, load и user testing.

## Главный принцип проекта

- Каждый `index.html` — это только вход в конкретную страницу.
- Вся логика интерфейса вынесена в модули.
- Основной код поддерживается в `TypeScript` внутри `src/`.
- Готовые браузерные модули лежат в `assets/scripts/`.
- Данные сайта и формы разделены на обычные JSON-файлы.

## Какая папка за что отвечает

- `index.html`, `about/index.html`, `news/index.html` и остальные HTML-файлы:
  только точки входа страниц. Они подключают стили, контейнер приложения и основной модуль.

- `src/`:
  исходники на `TypeScript`. Если проект дальше развивать, работать нужно в первую очередь здесь.

- `assets/scripts/`:
  готовая браузерная версия модулей, которую сейчас подключают страницы.

- `assets/styles/`:
  вся визуальная часть сайта.

- `data/`:
  контент сайта, который можно менять без переписывания интерфейса.

- `storage/`:
  локальная JSON-база заявок, куда сохраняются данные из форм.

- `server/server.ps1`:
  локальный сервер, который раздает страницы, API и принимает отправку форм.

- `tests/`:
  набор инструкций и скриптов для проверки проекта.

## Где лежат данные

- `data/site.json` — бренд, контакты, реквизиты, тексты.
- `data/animals.json` — каталог животных и характеристики карточек.
- `data/news.json` — новости.
- `data/faq.json` — ответы для окна `Задать вопрос`.
- `storage/adoption-applications.json` — анкеты на пристройство.
- `storage/help-requests.json` — заявки на помощь приюту.

## Как работают формы и JSON-база

- Форма `Хочу приютить` отправляет данные в `POST /api/adoption-applications`.
- Форма `Помощь приюту` отправляет данные в `POST /api/help-requests`.
- Сервер валидирует базовые поля и сохраняет записи в JSON-файлы из папки `storage/`.
- Админ-панель читает эти данные через:
  - `GET /api/adoption-applications`
  - `GET /api/help-requests`
- В админ-панели видны:
  - таблицы заявок;
  - последняя активность;
  - сырой JSON для наглядной проверки.

## Админ-панель

Админ-панель уже реализована. Она нужна, чтобы не просто говорить, что “данные сохраняются в JSON”, а реально показывать это в интерфейсе.

Как зайти в админ-панель:

1. Запустить сервер командой `powershell -ExecutionPolicy Bypass -File .\server\server.ps1`.
2. Открыть в браузере главную страницу `http://localhost:8080`.
3. Перейти вручную по адресу `/admin/` или открыть `http://localhost:8080/admin/`.
4. Если сервер запущен на другом порту, заменить `8080` на свой порт.
5. После отправки форм обновить страницу админ-панели, чтобы увидеть новые записи.

Открывается по адресу:

```text
http://localhost:8080/admin/
```

Если сервер запущен на другом порту, замените `8080` на свой порт, например:

```text
http://localhost:8086/admin/
```

Что там можно проверить:

- кто отправил анкету;
- телефон, email и комментарий;
- тип помощи или тип питомца;
- дату создания записи;
- сырой JSON из файлов `storage/*.json`.

Как показать это заказчику:

1. Открыть страницу `Хочу приютить` и отправить тестовую анкету.
2. Открыть страницу `Помощь приюту` и отправить тестовую заявку.
3. Сразу после этого открыть `http://localhost:8080/admin/`.
4. Показать, что новые записи появились в таблицах.
5. При необходимости открыть `storage/adoption-applications.json` и `storage/help-requests.json` и показать те же данные в сыром JSON.

## Как запустить проект

### Вариант 1. Быстрый запуск без Node.js

```powershell
cd C:\diana\lapa
powershell -ExecutionPolicy Bypass -File .\server\server.ps1
```

После этого сайт будет доступен по адресу:

```text
http://localhost:8080
```

Если порт занят, запустите сервер на другом порту:

```powershell
powershell -ExecutionPolicy Bypass -File .\server\server.ps1 -Port 8086
```

Тогда сайт откроется по адресу:

```text
http://localhost:8086
```

### Вариант 2. Если нужно пересобрать TypeScript-часть

```powershell
cd C:\diana\lapa
npm install
npm run build
```

Сборка идет из `src/` в `assets/scripts/`.

## Как быстро проверить, что JSON-база реально работает

1. Запустите сайт через `server/server.ps1`.
2. Откройте страницу `Хочу приютить` и отправьте тестовую анкету.
3. Откройте страницу `Помощь приюту` и отправьте тестовую заявку.
4. Перейдите в `/admin/`.
5. Убедитесь, что обе записи появились в таблицах.
6. При желании откройте файлы:
   - `storage/adoption-applications.json`
   - `storage/help-requests.json`

Именно это и есть локальная JSON-база проекта на данном этапе.

## Что важно понимать заказчику

- Проект уже имеет нормальную древовидную структуру, а не собран в один HTML-файл.
- HTML-файлы используются как входные точки страниц.
- Логику удобно расширять через `TypeScript`-модули.
- Контент удобно менять через `data/*.json`.
- Формы уже работают через локальное JSON-хранилище.
- Админ-панель уже показывает сохраненные записи в интерфейсе.
- Подключение реальной почты или полноценной БД можно сделать следующим этапом без слома текущей архитектуры.
