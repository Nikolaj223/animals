# Тестирование проекта

Этот файл нужен, чтобы заказчик или проверяющий мог самостоятельно развернуть проект и проверить, что сайт, формы и JSON-хранилище действительно работают.

В проекте есть 3 вида тестирования:

- `smoke test` — быстрая проверка основных страниц и API;
- `load test` — базовая нагрузочная проверка;
- `user testing` — ручной пользовательский сценарий.

## Подготовка

Перейдите в папку проекта:

```powershell
cd C:\diana\lapa
```

## 1. Запуск сайта

Основная команда:

```powershell
powershell -ExecutionPolicy Bypass -File .\server\server.ps1
```

Сайт откроется по адресу:

```text
http://localhost:8080
```

Если порт занят:

```powershell
powershell -ExecutionPolicy Bypass -File .\server\server.ps1 -Port 8086
```

Тогда адрес будет такой:

```text
http://localhost:8086
```

## 2. Smoke test

Команда:

```powershell
powershell -ExecutionPolicy Bypass -File .\tests\smoke-tests.ps1
```

Важно:

- smoke-тест сам поднимает временный сервер на `http://localhost:8090`;
- после завершения теста этот временный сервер автоматически останавливается;
- поэтому если открыть `http://localhost:8090` уже после завершения теста, страница может быть недоступна, и это нормально.

Если нужно, чтобы тестовый сервер остался работать после проверки:

```powershell
powershell -ExecutionPolicy Bypass -File .\tests\smoke-tests.ps1 -KeepServerRunning
```

Что проверяется:

- доступность основных HTML-страниц;
- доступность CSS и JS;
- работа API;
- базовая отправка тестовых форм.

Успешный результат:

- скрипт завершается без ошибки;
- в конце появляется список успешно проверенных маршрутов;
- видно, что POST-запросы к формам тоже прошли;
- в конце появляется пояснение, что временный сервер будет остановлен или оставлен запущенным.

## 3. Нагрузочный тест

Команда:

```powershell
powershell -ExecutionPolicy Bypass -File .\tests\load-test.ps1
```

Важно:

- load-тест тоже сам поднимает временный сервер, по умолчанию на `http://localhost:8091`;
- после завершения теста этот временный сервер автоматически останавливается;
- поэтому этот порт нужен только на время теста, а не для постоянного открытия сайта в браузере.

Если нужно оставить тестовый сервер после load-теста:

```powershell
powershell -ExecutionPolicy Bypass -File .\tests\load-test.ps1 -KeepServerRunning
```

Что показывает:

- количество виртуальных пользователей;
- общее число запросов;
- среднюю задержку;
- максимальную задержку;
- примерную пропускную способность.

Пример запуска с параметрами:

```powershell
powershell -ExecutionPolicy Bypass -File .\tests\load-test.ps1 -Port 8091 -Users 10 -RequestsPerUser 25
```

## 4. Пользовательское тестирование

Подробный ручной сценарий:

[user-testing.md](./user-testing.md)

Минимальный сценарий:

1. Запустите сервер.
2. Откройте сайт в браузере.
3. Пройдите шаги из `user-testing.md`.
4. После отправки форм обязательно откройте `/admin/`.
5. Убедитесь, что новые записи появились в интерфейсе и в JSON.

Что нужно проверить вручную:

- открывается главная страница;
- работает окно `Задать вопрос`;
- открывается каталог животных;
- работают фильтры каталога;
- открывается карточка животного;
- отправляется анкета `Хочу приютить`;
- отправляется форма `Помощь приюту`;
- открывается админ-панель;
- в админ-панели видны реальные данные из JSON;
- сайт корректно выглядит на мобильной ширине.

## Как проверить JSON-базу отдельно

После отправки форм откройте:

- `http://localhost:8080/admin/`
- или тот же адрес с вашим портом, например `http://localhost:8086/admin/`

Как зайти пошагово:

1. Запустите сервер.
2. Откройте главную страницу сайта.
3. В адресной строке перейдите на `/admin/`.
4. Если использовали другой порт, откройте `/admin/` на этом же порту.
5. Убедитесь, что в админ-панели видны таблицы и сырой JSON.

Также можно проверить сами файлы:

- `storage/adoption-applications.json`
- `storage/help-requests.json`

Если новая запись появилась и в админке, и в файле, значит JSON-база работает корректно.

## Полезные команды

Если нужно оставить тестовый сервер после smoke-теста:

```powershell
powershell -ExecutionPolicy Bypass -File .\tests\smoke-tests.ps1 -KeepServerRunning
```

Если нужно запустить сайт на другом порту:

```powershell
powershell -ExecutionPolicy Bypass -File .\server\server.ps1 -Port 8087
```

Если нужно пересобрать TypeScript:

```powershell
npm install
npm run build
```

Если нужно остановить зависший PowerShell-процесс:

```powershell
Get-Process powershell
Stop-Process -Id <PID> -Force
```

## Что показать заказчику

Минимальная демонстрация:

1. Запустить сервер.
2. Открыть главную страницу.
3. Показать каталог и карточку животного.
4. Заполнить анкету `Хочу приютить`.
5. Заполнить форму помощи приюту.
6. Открыть `/admin/` и показать, что записи реально отобразились.
7. При необходимости открыть `storage/*.json` и показать сырой JSON.
